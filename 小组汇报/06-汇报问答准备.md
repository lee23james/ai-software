# 06 汇报问答准备

## Q1：你们是否为项目前端设计了纸质原型 / 线框图 / 高保真模型 / 可操作原型？

**回答：**

我们当前已经完成了一个可操作的低保真前端原型。用户可以在浏览器中访问登录页、首页看板、岗位列表、岗位详情、数据分析、简历辅助和后台管理等页面。当前原型主要用于确认页面流程、功能边界和系统交互，暂时还没有制作高保真 Figma 模型。后续可以在现有页面基础上补充高保真视觉设计。

## Q2：你们前端的 MVC 组件是什么？

**回答：**

我们前端采用 Vue3 组件化架构，不是传统后端 MVC。对应关系上，Model 由 Pinia 状态管理和 API 请求模块承担，View 由 Vue 页面组件和布局组件承担，Controller / ViewModel 由 Vue Router、页面事件处理和接口调用逻辑承担。后端则采用 Spring Boot 三层架构，Controller 负责接口，Service 负责业务逻辑，Mapper 负责数据访问。

## Q3：你们评估 / 测试项目用户体验的计划是什么？

**回答：**

我们计划通过三类方式评估用户体验。第一是功能流程测试，验证学生用户能否顺利完成岗位浏览、筛选、详情查看、简历上传和岗位匹配。第二是邀请同学进行试用，收集他们对页面易用性、信息清晰度和匹配建议可理解性的反馈。第三是根据典型用户场景进行测试，例如“想找 Java 实习的学生”“不清楚就业方向的学生”“需要简历优化建议的学生”，观察他们是否能顺利完成核心任务。

## Q4：你们项目的组件图是怎样的？

**回答：**

系统组件主要分为 Vue 前端应用、Spring Boot 后端服务、FastAPI AI 服务和数据存储四部分。前端负责页面展示、路由跳转、状态管理和图表展示；后端负责用户、岗位、数据分析、简历、后台管理等业务接口；AI 服务负责简历解析、岗位匹配和优化建议；数据库负责保存用户、岗位、简历、收藏和浏览记录等数据。组件之间通过 HTTP JSON 接口通信。

## Q5：你们项目的部署图是怎样的？

**回答：**

当前开发环境采用本地三端部署。前端 Vue 服务运行在 5173 端口，Spring Boot 后端运行在 8080 端口，FastAPI AI 服务运行在 8000 端口。后端当前默认使用 H2 内存数据库，方便团队成员无需安装 MySQL 就可以启动项目。未来正式部署时，前端可以部署到 Nginx，后端和 AI 服务作为独立服务运行，数据库切换为 MySQL，Redis 作为可选缓存。

## Q6：你们项目的包图是怎样的？

**回答：**

后端包结构包括 controller、service、service.impl、mapper、entity、dto、vo、common、config、exception，对应接口层、业务层、数据访问层、数据模型和通用配置。前端包结构包括 views、layouts、router、stores、api、assets，对应页面、布局、路由、状态管理和接口请求。AI 服务包结构包括 routers、schemas、services，对应接口路由、请求模型和 AI 业务逻辑。

## Q7：是否有使用虚拟化技术的计划？

**回答：**

当前开发阶段暂时没有强制使用虚拟化技术，因为项目可以直接在本地通过 Node.js、Java、Python 三个环境启动。后续如果需要统一团队环境或进行部署，可以使用 Docker 将前端、后端、AI 服务和 MySQL 分别容器化，再通过 Docker Compose 统一编排。这样可以减少不同电脑环境导致的问题，也便于后续部署。

## Q8：项目中是否存在潜在的安全风险？如何降低？

**回答：**

项目存在的潜在安全风险主要包括用户密码泄露、接口越权访问、上传恶意简历文件、SQL 注入、AI 服务异常返回等。降低方式包括：后端使用 JWT 做登录鉴权和角色权限控制；密码加密存储；接口参数校验；文件上传限制格式和大小；数据库操作使用 MyBatis-Plus 参数化查询；AI 服务调用设置超时、异常兜底和返回格式校验，避免 AI 服务异常影响主系统。

## Q9：未来产品的峰值需求 / 容量是多少？

**回答：**

作为课程项目，我们的初始目标是支持班级或学院范围内的演示和试用。预估初期数据量为数千到数万条岗位数据，用户规模为几十到几百人，峰值并发主要集中在课堂演示或集中测试时。后续如果扩展到真实学院就业平台，可以按更多用户和更多岗位数据进行容量规划，例如十万级岗位数据、千级用户、几十到上百并发访问。

## Q10：是否存在性能瓶颈？如何改进？

**回答：**

潜在性能瓶颈主要包括岗位列表多条件查询、数据分析统计计算、ECharts 大数据渲染、AI 简历匹配调用耗时和文件上传处理。改进方式包括：数据库字段加索引；分页查询岗位列表；将统计结果预计算并缓存；前端图表按需加载和分页展示；AI 服务设置超时和异步处理；常用匹配结果缓存；后续可以使用 Redis 缓存热点数据，必要时将 AI 服务独立扩容。
