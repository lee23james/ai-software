# 项目立项说明

## 一、项目名称

**面向大学生就业的招聘数据分析与智能求职辅助平台**

---

## 二、项目背景与意义

随着高校毕业生数量逐年增加，大学生在求职过程中普遍面临以下问题：

1. 缺乏对就业市场的整体认知，不清楚不同岗位、城市和专业的薪资水平与需求趋势。
2. 招聘信息分散，筛选成本高，难以快速找到适合自己的岗位。
3. 缺乏针对岗位的简历优化能力，导致求职效率较低。

因此，本项目旨在构建一个集 **招聘数据分析、可视化展示、AI 智能辅助** 于一体的平台，帮助大学生更好地进行就业决策，同时也为教师或辅导员提供就业趋势分析工具。

---

## 三、项目目标

本项目的核心目标如下：

- 构建一个可运行的招聘数据分析平台，形成完整的前后端系统。
- 实现招聘数据的结构化管理与可视化分析。
- 提供基础的智能求职辅助能力，例如简历解析、岗位匹配、简历优化建议等。
- 支持用户登录、注册与权限管理。
- 为学生提供岗位检索、岗位收藏、简历上传、岗位匹配等功能。
- 为教师或辅导员提供就业趋势分析与数据看板。
- 为管理员提供岗位数据维护、用户管理、系统数据清理等后台管理能力。

## 四、目标用户

平台主要面向以下三类用户。

### 4.1 大学生用户

主要需求包括：

- 查看招聘岗位信息。
- 按城市、岗位、学历、薪资、技能等条件筛选岗位。
- 查看岗位详情。
- 上传个人简历。
- 获取岗位匹配结果。
- 获得简历优化建议。
- 收藏感兴趣岗位。
- 查看历史浏览记录。

### 4.2 教师 / 辅导员用户

主要需求包括：

- 查看就业市场整体趋势。
- 分析不同城市、岗位、专业方向的就业需求。
- 查看薪资水平、岗位数量、学历要求等统计结果。
- 辅助开展就业指导工作。

### 4.3 管理员用户

主要需求包括：

- 管理用户信息。
- 管理岗位数据。
- 导入、清洗或重置招聘数据。
- 查看系统运行状态和后台数据看板。
- 维护系统基础配置。

## 六、系统架构设计

本系统采用前后端分离架构，并将 AI 服务作为独立模块进行设计。

整体架构分为四层：

1. 表现层：前端页面与可视化展示。
2. 业务服务层：Java 后端业务接口。
3. AI 服务层：Python 智能分析服务。
4. 数据层：MySQL、Redis、文件存储等。

### 6.1 表现层：前端系统

技术选型：

- Vue 3
- Element Plus
- Vue Router
- Pinia
- Axios
- ECharts

主要功能：

- 用户登录与注册页面。
- 系统首页。
- 岗位列表页。
- 岗位详情页。
- 数据分析仪表盘。
- 简历上传页面。
- 收藏管理页面。
- 历史浏览记录页面。
- 管理员后台页面。

### 6.2 业务服务层：后端系统

技术选型：

- Spring Boot
- MyBatis / MyBatis-Plus
- MySQL
- JWT
- Swagger / Knife4j

主要功能：

- 用户管理。
- 登录鉴权。
- 权限管理。
- 岗位信息管理。
- 收藏记录管理。
- 浏览历史管理。
- 简历文件管理。
- 数据分析接口。
- 管理员后台接口。
- 与 AI 服务进行 HTTP 调用。

建议采用三层架构：

- Controller：提供 RESTful API。
- Service：处理业务逻辑。
- Mapper / DAO：负责数据库访问。

同时补充以下结构：

- Entity：数据库实体对象。
- DTO：请求数据对象。
- VO：响应展示对象。
- Common Result：统一返回格式。
- Global Exception Handler：全局异常处理。
- JWT Filter：登录鉴权过滤器。

### 6.3 AI 服务层

技术选型：

- Python
- FastAPI 或 Flask

主要功能：

- 简历解析。
- 岗位匹配计算。
- 文本生成。
- 简历优化建议生成。
- 岗位 JD 分析。
- 技能关键词提取。

AI 服务与 Java 后端之间通过 HTTP 接口通信。

### 6.4 数据层

数据层包括：

- MySQL：业务数据存储。
- Redis：缓存，可选。
- 文件存储：用于保存上传的简历文件。
- 预计算统计结果表：用于支撑可视化展示。

### 6.5 架构特点

- 分层清晰，符合软件工程设计规范。
- 模块解耦，便于扩展 AI 能力。
- 便于团队成员分工协作。
- 前端、后端、AI、数据分析职责边界明确。
- 适合课程设计、系统演示和答辩展示。
