# 接口说明

## 1. 统一返回格式

后端接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

AI 服务接口返回结构保持简洁，后续由 Java 后端统一封装给前端。

## 2. 后端接口

### 2.1 健康检查

```http
GET /api/health
```

示例响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "service": "backend",
    "status": "UP"
  }
}
```

### 2.2 岗位列表

```http
GET /api/job/list
```

当前返回模拟数据，后续接入数据库分页查询。

### 2.3 城市岗位统计

```http
GET /api/analysis/city-job-count
```

当前返回 ECharts 可直接消费的基础数组。

### 2.4 管理员看板

```http
GET /api/admin/dashboard
```

当前返回岗位数、用户数、简历数等模拟统计值。

## 3. AI 服务接口

### 3.1 健康检查

```http
GET /health
```

### 3.2 简历解析

```http
POST /ai/resume/parse
```

请求体：

```json
{
  "resumeText": "熟悉 Java、Spring Boot、Vue 和 MySQL。"
}
```

### 3.3 岗位匹配

```http
POST /ai/job/match
```

请求体：

```json
{
  "resumeText": "熟悉 Java、Spring Boot、Vue 和 MySQL。",
  "jobDescription": "要求掌握 Java、Spring Boot、MySQL，了解 Redis。"
}
```

### 3.4 简历优化建议

```http
POST /ai/resume/suggestion
```

请求体：

```json
{
  "resumeText": "做过课程设计。",
  "targetJob": "Java 后端开发实习生"
}
```
