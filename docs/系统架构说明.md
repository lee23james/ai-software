# 系统架构说明

## 1. 目标

本项目采用前后端分离架构，拆分为前端、Java 后端、Python AI 服务三个可独立启动的子项目。当前阶段只搭建可运行基础框架，业务细节由后续开发成员继续完善。

## 2. 顶层目录

```text
ai-software
├── frontend/          # Vue 3 前端工程
├── backend/           # Spring Boot 后端工程
├── ai-service/        # FastAPI AI 服务工程
├── docs/              # 项目文档
├── data/              # 示例数据与导入模板
├── scripts/           # 启动脚本
├── 项目立项说明.md
└── README.md
```

## 3. 子系统边界

### 3.1 前端系统

前端负责页面展示、路由跳转、状态管理、接口调用和图表渲染。

技术栈：

- Vue 3
- Vite
- Element Plus
- Vue Router
- Pinia
- Axios
- ECharts

### 3.2 后端系统

后端负责用户、岗位、收藏、简历、分析接口、管理员后台等核心业务。当前阶段先提供健康检查和模拟数据接口，保证前端可以联调。

技术栈：

- Spring Boot
- Spring MVC
- MyBatis-Plus
- H2 本地内存数据库
- MySQL 预留

说明：基础框架默认使用 H2，避免开发初期必须安装 MySQL。后续接入 MySQL 时，只需要修改 `backend/src/main/resources/application.yml` 中的数据源配置。

### 3.3 AI 服务

AI 服务负责简历解析、岗位匹配和简历优化建议。当前阶段先提供模拟返回，保证接口协议稳定。

技术栈：

- Python
- FastAPI
- Uvicorn
- Pydantic
- Pytest

## 4. 服务端口

| 服务 | 默认端口 | 启动目录 |
| --- | --- | --- |
| 前端 | 5173 | `frontend` |
| 后端 | 8080 | `backend` |
| AI 服务 | 8000 | `ai-service` |

## 5. 数据流

```text
浏览器
  ↓
Vue 前端
  ↓ HTTP /api/*
Spring Boot 后端
  ↓ HTTP /ai/*
FastAPI AI 服务
  ↓
返回结构化 JSON
```

前端不直接访问 AI 服务，AI 能力统一由 Java 后端转发或封装，便于权限控制、日志记录、容错和缓存。

## 6. 当前框架范围

当前框架包含：

- 前端基础页面和路由。
- 后端基础包结构、统一返回格式和模拟接口。
- AI 服务基础路由和模拟结果。
- 示例招聘数据。
- 启动脚本。
- 接口说明文档。

当前框架暂不包含：

- 真实登录鉴权。
- 真实 MySQL 表结构迁移。
- 真实简历解析算法。
- 真实岗位匹配算法。
- 完整后台管理逻辑。
