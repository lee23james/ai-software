# 数据库设计说明（已落地）

本项目数据库已按“大学生就业招聘数据分析与智能求职辅助平台”重构为业务化模型。

## 1. 设计目标

- 支撑岗位数据导入、查询、详情、分析。
- 支撑统一账号、学生档案、简历解析、岗位匹配。
- 支撑收藏、投递、后台导入审计和操作日志。
- 本地与协作环境默认使用 **MySQL 8**（见 `application.yml`）；`schema.sql` 仅作备用参考。

## 2. 核心业务表

- 用户与画像：`sys_user`、`student_profile`
- 公司与岗位：`company_info`、`job_info`
- 技能体系：`skill_dict`、`job_skill`、`resume_skill`
- 简历体系：`resume_info`、`resume_parse_result`
- 求职行为：`job_favorite`、`job_application`
- 匹配与分析：`job_match_record`、`analysis_metric_snapshot`
- 数据治理：`data_import_batch`、`operation_log`

## 3. 关键索引与约束

- 用户：`uk_sys_user_username`、`idx_sys_user_role_status`
- 岗位：`idx_job_info_city_status`、`idx_job_info_salary`、`idx_job_info_publish_time`
- 技能关联：`uk_job_skill_job_skill`、`uk_resume_skill_resume_skill`
- 匹配记录：`idx_job_match_record_resume_job`、`idx_job_match_record_score`
- 导入与审计：`uk_data_import_batch_no`、`idx_operation_log_module_type`

## 4. 状态字段约定

- 通用启停：`1` 启用/正常，`0` 禁用/无效
- `job_info.status`：`0` 下架，`1` 上架，`2` 待审核
- `resume_info.parse_status`：`0` 未解析，`1` 解析中，`2` 成功，`3` 失败
- `job_application.application_status`：`submitted`、`viewed`、`interview`、`rejected`、`accepted`

## 5. 脚本位置

- 应用启动时自动执行（`spring.sql.init`）：`backend/src/main/resources/sql/schema-mysql.sql`
- 备用 / 对照：`backend/src/main/resources/schema.sql`（非默认数据源使用）

## 6. 实施建议

1. 业务代码直接对齐目标表。
2. 新增接口优先复用技能字典和关联表，避免 `skill_tags` 单字段膨胀。
3. 统计类查询优先聚合 `job_info + job_skill`，高频指标写入 `analysis_metric_snapshot`。
